## 使用方法

### 1. 开启调试模式
点击页面底部的 **🔧 调试模式** 按钮，会显示：
- 实时加速度值
- X/Y/Z三轴数据
- 摇动状态
- 测试按钮

### 2. 测试摇一摇
- **方法A**：摇晃手机（强烈晃动）
- **方法B**：点击调试面板中的"测试摇一摇"按钮

### 3. 调试信息说明
- **加速度**：总加速度值（√(x² + y² + z²)）
- **X/Y/Z**：三个轴的加速度分量
- **状态**：
  - "等待摇动..." - 等待用户摇动
  - "检测到摇动！" - 检测到超过阈值的晃动

### 4. 调试日志
打开微信开发者工具的控制台，可以看到：
- `加速度变化:` - 传感器实时数据
- `检测到摇一摇！加速度:` - 触发日志
- `调试模式: true/false` - 模式切换日志
- `防抖已解除，可以再次摇动` - 防抖解除日志

## 常见问题

### Q1: 摇一摇还是没有反应？
**解决方法：**
1. 开启调试模式，查看加速度数据是否有变化
2. 尝试更强烈地摇晃手机（上下左右快速晃动）
3. 点击"测试摇一摇"按钮，确认功能是否正常

### Q2: 加速度数据一直是0？
**可能原因：**
- 设备不支持加速度传感器（旧设备）
- 微信小程序没有获取传感器权限
- 需要真机测试（开发工具可能不支持）

### Q3: 阈值太高/太低？
**调整方法：**
在 `pages/index/index.js` 中修改 `SHAKE_THRESHOLD` 常量：
```javascript
const SHAKE_THRESHOLD = 10;  // 当前值，可调整范围 5-20
```
- `5` - 非常敏感
- `10` - 适中（推荐）
- `15` - 较难触发
- `20` - 很难触发

### Q4: 防抖时间太长/太短？
**调整方法：**
在 `pages/index/index.js` 中修改 `shakeThreshold`：
```javascript
shakeThreshold: 3000  // 当前值：3秒，可改为 1000-5000ms
```

### Q5: 页面切换后摇一摇不工作？
**原因：**页面隐藏时会自动关闭传感器监听以节省电量
**解决：** 回到页面后会自动重新开启

```javascript
acceleration = Math.sqrt(x*x + y*y + z*z)
if (acceleration > SHAKE_THRESHOLD) {
  // 触发算卦
}
```

## 性能优化

### 自动开关监听
- **页面显示时**：自动开启监听
- **页面隐藏时**：自动关闭监听（节省电量）

### 防抖机制
- 默认 3 秒间隔
- 避免频繁触发算卦

### 加载优化
- 启用 `lazyCodeLoading: true` 提升加载速度

## 建议使用方式
