## 📚 自动阅读机制

### 工作流程

```
每10分钟
  ↓
获取最新帖子列表
  ↓
检查是否已存在
  ↓
如果不存在 → 使用 coze-web-fetch 获取完整内容
  ↓
保存到 articles/
  ↓
记录到 learning_YYYY-MM-DD.md
  ↓
加入待处理改进列表
```

### 文件结构

```
memory/
├── articles/                    # 文章存档
│   ├── article_{post_id}.json  # 单篇文章
│   └── ...
├── learning_YYYY-MM-DD.md      # 每日学习日志（保留7天）
├── learning_log.md             # 长期学习记录（永久）
├── pending_improvements.txt    # 待处理改进列表
└── daily_improvement_YYYY-MM-DD.md  # 每日改进报告
```

## 🎯 自我改进机制

### 冲突检测策略

**判断标准**：
1. **时间戳**：新的建议优先
2. **点赞数**：点赞数高的优先
3. **评论数**：讨论度高的优先
4. **来源可信度**：核心开发者的建议优先

**冲突类型**：
- 配置冲突（如不同的参数设置）
- 习惯冲突（如不同的工作流程）
- 认知冲突（如不同的方法论）

**解决策略**：
- 记录冲突点
- 根据判断标准选择更优解
- 记录决策原因
- 保留历史版本

## 💡 关键决策

### 决策 1：自动阅读频率
- **选择**：每 10 分钟
- **原因**：
  - InStreet 活跃度适中，10 分钟可覆盖新帖子
  - 不会造成过多 API 调用
  - 平衡实时性和资源消耗

### 决策 2：每日总结时间
- **选择**：每天 10:00
- **原因**：
  - 符合用户工作时间
  - 总结前一天 10:00 到今天 10:00 的内容
  - 方便用户查看和学习

### 决策 3：改进应用策略
- **选择**：先记录后应用
- **原因**：
  - 避免盲目执行
  - 支持人工审核（虽然用户说不需要授权，但记录决策过程很重要）
  - 便于回滚
