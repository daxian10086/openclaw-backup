# mootdx 数据更新机制说明

## 📊 数据来源和更新方式

### mootdx 的工作原理

```
通达信软件
    ↓ （每天收盘后下载数据）
本地数据文件（.day 格式）
    ↓ （mootdx 读取）
Python 数据
```

**核心结论**: mootdx **不是实时获取**数据，而是读取通达信本地数据。

---

## 🔍 详细说明

### 1. 数据来源

| 项目 | 说明 |
|------|------|
| **数据来源** | 通达信本地数据文件 |
| **文件格式** | .day（日线）、.lc1（1分钟）、.ln1（5分钟）等 |
| **存储位置** | 通达信安装目录下的 vipdoc/ 文件夹 |
| **数据获取方式** | 读取本地文件，不是网络请求 |

### 2. 数据更新时机

**通达信数据更新流程**：

```
每天 15:00 收盘后
    ↓
通达信自动下载数据
    ↓
数据保存到本地文件
    ↓
mootdx 可以读取到最新数据
```

**重要**: mootdx **本身不会自动更新数据**，它只是读取通达信已经下载的数据。

### 3. 实时性分析

| 工具 | 数据来源 | 实时性 | 更新方式 |
|------|---------|--------|---------|
| **mootdx** | 通达信本地 | ⚠️ 非实时 | 依赖通达信更新 |
| **AKShare** | 网络API | ✅ 实时 | 自动从网络获取 |

**对比**:

| 场景 | mootdx | AKShare |
|------|--------|---------|
| **15:00 当天收盘数据** | ✅ 可以读取（通达信已下载） | ✅ 可以获取 |
| **14:30 盘中数据** | ❌ 不能（通达信未更新） | ✅ 可以获取 |
| **历史数据** | ✅ 快速（本地读取） | ✅ 可以获取（网络请求） |
| **离线使用** | ✅ 支持 | ❌ 不支持 |

## 💡 使用建议

### 适用场景

#### ✅ 适合使用 mootdx：

1. **收盘后分析**
   - 每天 15:00 之后
   - 通达信已经下载了最新数据

2. **离线分析**
   - 没有网络连接时
   - 需要批量处理历史数据

3. **历史回测**
   - 需要大量历史数据
   - 本地读取速度快

4. **通达信用户**
   - 已经使用通达信软件
   - 习惯通达信的数据格式

#### ❌ 不适合使用 mootdx：

1. **盘中实时分析**
   - 需要盘中实时数据
   - 建议使用 AKShare

2. **分钟级数据**
   - 需要实时的分钟数据
   - 建议使用 AKShare

3. **没有通达信**
   - 未安装通达信软件
   - 建议直接使用 AKShare

## 🔧 数据更新流程

### 如果你使用通达信

**自动更新**（推荐）：

1. 打开通达信软件
2. 通达信每天收盘后自动下载数据
3. mootdx 读取最新数据

**手动更新**：

1. 打开通达信
2. 点击「数据」→「日线数据下载」
3. 选择股票和时间范围
4. 点击「开始下载」

### 如果你需要实时数据

**使用 AKShare**：

```python
import akshare as ak

# 获取实时行情
df = ak.stock_zh_a_spot_em()

# 获取历史数据
df = ak.stock_zh_a_hist(symbol='600396', period='daily')
```

## 📝 总结

### mootdx 特点

✅ **优点**：
- 离线可用
- 读取速度快
- 数据完整（如果通达信数据完整）
- 适合批量处理

❌ **缺点**：
- 需要通达信软件
- 不是实时数据
- 依赖通达信更新
- 安装门槛较高

### AKShare 特点

✅ **优点**：
- 实时数据
- 无需其他软件
- 使用简单
- 数据来源广泛

❌ **缺点**：
- 需要网络连接
- 网络请求速度较慢
- 依赖第三方API稳定性

### 最终建议

| 你的情况 | 推荐工具 |
|---------|---------|
| 已使用通达信 + 收盘后分析 | mootdx ⭐ |
| 需要实时数据 | AKShare ⭐ |
| 没有通达信 | AKShare ⭐ |
| 需要离线分析 | mootdx ⭐ |
| 综合需求 | 两个都用 🎯 |

---

**记住**: mootdx 是读取本地数据，不是实时获取！
