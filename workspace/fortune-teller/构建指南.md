# 周易算卦 APK 构建指南

## 快速开始（3分钟）

### 前置条件
- 电脑已安装 Android Studio（最新版本）
- 电脑连接互联网（下载SDK依赖）

---

## 方法一：使用 Android Studio 构建（推荐）

### 步骤1：下载并解压项目
1. 从飞书云盘下载 `周易算卦-APK项目.tar.gz`
2. 解压到任意文件夹
3. 进入解压后的 `fortune-teller` 文件夹

### 步骤2：用 Android Studio 打开
1. 打开 Android Studio
2. 点击 `File` → `Open`
3. 选择 `fortune-teller` 文件夹
4. 点击 `OK`

### 步骤3：等待 Gradle 同步
- 第一次打开会自动下载依赖（约3-10分钟）
- 底部显示 "Gradle sync finished" 表示完成

### 步骤4：构建 Release APK
1. 菜单栏选择 `Build` → `Generate Signed Bundle / APK`
2. 选择 `APK`，点击 `Next`
3. 配置签名密钥（首次需要创建）：

   **创建新密钥：**
   - 点击 `Create new...`
   - 设置密钥库路径（如：`my-key.jks`）
   - 设置密码（记住密码！）
   - 设置别名（如：`key0`）
   - 设置别名密码
   - 填写至少一项信息（如：CN=MyApp）

4. 选择 `release` 构建类型
5. 点击 `Finish` 开始构建
6. 等待构建完成（约1-3分钟）

### 步骤5：找到 APK
APK 文件位置：
```
fortune-teller/app/build/outputs/apk/release/app-release.apk
```

### 步骤6：安装到手机
1. 将 APK 文件传输到手机
2. 在手机上开启"允许安装未知来源应用"
3. 点击 APK 文件安装

---

## 方法二：使用命令行构建

### 步骤1：打开终端
在 Android Studio 中：`View` → `Tool Windows` → `Terminal`

### 步骤2：构建 Release APK
```bash
# 进入项目目录
cd fortune-teller

# 清理之前的构建
./gradlew clean

# 构建 Release APK
./gradlew assembleRelease
```

### 步骤3：找到 APK
```
app/build/outputs/apk/release/app-release.apk
```

---

## 方法三：构建 Debug APK（不需要签名）

### 快速测试版本
```bash
./gradlew assembleDebug
```

APK 位置：`app/build/outputs/apk/debug/app-debug.apk`

**注意**：Debug APK 不需要签名，但安装时可能提示"来自未知来源"

## 常见问题

### Q1: Gradle 同步失败
**解决方案：**
- 检查网络连接
- 关闭 VPN/代理
- File → Invalidate Caches → Invalidate and Restart

### Q2: 构建时提示 SDK 未安装
**解决方案：**
- Tools → SDK Manager
- 确保勾选：
  - Android 14.0 (API 34)
  - Android SDK Build-Tools 34.0.0
- 点击 Apply 安装

### Q3: 找不到 ./gradlew 文件
**解决方案：**
- File → Settings → Build, Execution, Deployment → Gradle
- 勾选 "Use Gradle wrapper"
- 点击 OK 后重启 Android Studio

### Q4: 构建后找不到 APK
**解决方案：**
1. 在 Android Studio 左侧项目视图中
2. 切换到 "Project" 视图（而不是 "Android" 视图）
3. 导航到：`app/build/outputs/apk/release/`
4. 或使用命令查找：
   ```bash
   find . -name "*.apk" -type f
   ```

### Q5: APK 安装失败
**可能原因：**
- 签名问题：使用正确的签名密钥
- 版本冲突：卸载旧版本后再安装
- 权限问题：开启"允许安装未知来源应用"

## APK 签名说明

### 为什么需要签名？
- Android 要求所有 APK 必须签名才能安装
- Release APK 必须使用你自己的签名密钥
- Debug APK 会自动使用 Android Studio 提供的调试签名

### 保存好你的签名密钥！
⚠️ **重要**：
- 密钥库文件（.jks）必须安全保存
- 如果丢失密钥，将无法更新已发布的 APK
- 建议备份到多个安全位置

## 版本管理

### 修改版本号
打开 `app/build.gradle`：
```gradle
defaultConfig {
    applicationId "com.fortunetelling.i.ching"
    versionCode 1    // 每次更新 +1
    versionName "1.0"  // 用户看到的版本号
}
```

### 更新 APK 后
- `versionCode` 必须增加
- 用户安装新版本时会自动替换旧版本
- 如果使用相同签名，用户数据会保留
