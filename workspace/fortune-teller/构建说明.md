# 周易算卦 APK - 快速构建指南

## 🚀 最简单的构建方法（推荐）

### Windows 用户
1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. 用 Android Studio 打开 `fortune-teller` 文件夹

## ⚡ 一键构建脚本（需要先安装 Android SDK）

如果已经安装了 Android SDK，可以使用一键构建脚本：

```bash
# 进入项目目录
cd fortune-teller

# 给脚本添加执行权限
chmod +x setup-and-build.sh

# 运行构建脚本
./setup-and-build.sh release
```

脚本会自动：
1. 检查 Java 环境
2. 检查 Android SDK
3. 下载 Gradle（如果需要）
4. 清理旧构建
5. 构建 APK

## 🌐 在线构建（无需安装任何工具）

### GitHub Actions 方法（最简单）

#### 步骤：
1. 创建 GitHub 账号
2. 创建新仓库
3. 上传项目所有文件
4. 进入 `Actions` 标签
5. 选择 `Build Android APK`
6. 点击 `Run workflow`
7. 等待 5-10 分钟
8. 下载构建的 APK

#### 详细步骤：

1. **创建 GitHub 仓库**
   - 访问 https://github.com/new
   - 仓库名：`i-ching-fortune-teller`
   - 选择 `Public`
   - 点击 `Create repository`

2. **上传项目文件**
   - 点击 `uploading an existing file`
   - 选择项目所有文件
   - 或者直接拖拽文件夹
   - 点击 `Commit changes`

3. **触发自动构建**
   - 进入仓库
   - 点击 `Actions` 标签
   - 选择 `Build Android APK`
   - 点击 `Run workflow` → `Run workflow`
   - 等待构建完成（约 5-10 分钟）

4. **下载 APK**
   - 构建完成后，点击进入构建任务
   - 滚动到 `Artifacts` 部分
   - 下载 `app-debug.apk` 或 `app-release.apk`

## ❓ 常见问题

### Q1: Android Studio 同步很慢怎么办？
**A:**
- 检查网络连接
- 关闭 VPN
- 使用国内镜像源（修改 `build.gradle`）

### Q2: 构建失败提示找不到 Android SDK？
**A:**
1. 打开 Android Studio
2. Tools → SDK Manager
3. 确保勾选：
   - Android 14.0 (API 34)
   - Android SDK Build-Tools 34.0.0
4. 点击 Apply 安装

### Q3: APK 安装失败？
**A:**
- 开启"允许安装未知来源应用"
- 卸载旧版本后重新安装
- 检查手机系统版本（需 Android 7.0+）

### Q4: 构建时内存不足？
**A:**
1. 关闭其他应用
2. 在 `gradle.properties` 中添加：
   ```properties
   org.gradle.jvmargs=-Xmx4096m -XX:MaxMetaspaceSize=512m
   org.gradle.daemon=true
   org.gradle.parallel=true
   ```

### Q5: GitHub Actions 构建失败？
**A:**
- 确保所有文件都已上传
- 检查 `.github/workflows/build.yml` 文件是否存在
- 查看 Actions 日志获取详细错误
