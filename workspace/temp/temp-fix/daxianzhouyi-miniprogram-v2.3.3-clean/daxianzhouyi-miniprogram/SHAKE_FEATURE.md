# 摇一摇算卦功能说明（v2.0.9 优化版）

## 功能流程

### 新交互流程（已优化）

```
┌─────────────────────────────────────────────────────────────┐
│  1. 点击"算一卦"按钮                                         │
│     ↓                                                       │
│  2. 显示提示："📱 摇动手机开始算卦"                          │
│     ↓                                                       │
│  3. 用户开始摇晃手机                                         │
│     ↓                                                       │
│  4. 显示"正在摇卦..."动画（持续显示，直到停止摇动）          │
│     ↓                                                       │
│  5. 用户停止摇动（800ms内无摇动视为停止）                    │

│     ↓                                                       │
│  7. 显示算卦结果                                             │
└─────────────────────────────────────────────────────────────┘
```

### 状态说明

| 状态 | 显示内容 | 触发条件 |
|------|---------|---------|
| 等待摇动 | "📱 摇动手机开始算卦" | 点击按钮后，用户未开始摇动 |
| 正在摇卦 | "正在摇卦..." + 动画 | 检测到用户正在摇动手机 |
| 算卦中 | "算卦中..." + 动画 | 用户停止摇动后，显示2秒 |
| 结果显示 | 卦象信息 + 运势分析 | 算卦完成后 |

## 技术实现

### 摇动检测逻辑

1. **开始检测**：点击按钮后启动加速度监听
2. **摇动检测**：使用欧几里得距离计算加速度变化
   ```javascript
   const acceleration = Math.sqrt(deltaX² + deltaY² + deltaZ²)
   ```
3. **停止检测**：800ms内未检测到超过阈值的加速度变化，视为停止摇动

### 核心代码

```javascript
// 监听加速度变化
handleAccelerometerChange(res) {
  const { x, y, z } = res;
  const acceleration = 计算加速度变化量;

  if (acceleration > threshold) {
    // 检测到摇动
    this.lastShakeTime = now;

    if (!this.data.isShaking) {
      // 首次摇动，显示"正在摇卦..."
      this.setData({ isShaking: true, shakeStatusText: '正在摇卦...' });
    }

    // 重置停止检测定时器
    clearTimeout(this._stopShakeTimer);
    this._stopShakeTimer = setTimeout(() => {
      this.onShakeStopped(); // 800ms后无摇动，视为停止
    }, 800);
  }
}

// 摇动停止处理
onShakeStopped() {
  // 1. 停止监听
  this.stopAccelerometer();

  // 2. 显示"算卦中..."
  this.setData({ isShaking: false, isAnimating: true });

  // 3. 2秒后显示结果
  setTimeout(() => this.performCastHexagram(), 2000);
}
```

## 参数配置

| 参数 | 值 | 说明 |
|------|-----|------|
| shakeThreshold | 0.8 m/s² | 摇动检测阈值 |
| stopShakeDelay | 800ms | 停止摇动的检测延迟 |
| autoStartDelay | 8000ms | 自动开始算卦的超时时间 |

## 用户体验优化

1. **实时反馈**：用户摇动时立即看到"正在摇卦..."，知道系统已响应
2. **自然过渡**：停止摇动后，自动进入"算卦中..."，无需额外操作
3. **容错设计**：8秒未摇动自动开始，防止用户不知道如何操作
4. **资源管理**：页面切换时自动停止监听，节省电量

## 测试建议

1. **正常流程测试**
   - 点击按钮 → 看到提示 → 摇手机 → 看到"正在摇卦..." → 停止 → 看到"算卦中..." → 显示结果

2. **快速摇动测试**
   - 快速摇动后立即停止，验证是否能正确检测停止

3. **长时间摇动测试**
   - 持续摇动3-5秒，验证"正在摇卦..."是否持续显示

4. **不摇动测试**
   - 点击按钮后不摇动，等待8秒，验证是否自动开始

## 注意事项

1. 摇一摇功能需要手机支持加速度传感器
2. 首次使用可能需要在微信中授权传感器权限
3. 建议在真机上测试，模拟器无法模拟摇一摇
4. 如果摇动检测不灵敏，可以调低 `shakeThreshold` 值
